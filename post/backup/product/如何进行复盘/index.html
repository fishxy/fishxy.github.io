关于复盘

最近参加公司了公司关于"复盘"方法论的培训，相对全面的跟同学和老师一起讨论了关于"复盘"的一些问题，受益匪浅。而且经过自己跟团队人员的实际践行，越来越觉得"复盘"是帮助个人和团队共同成长的高效工具。

这里我想提一个有关复盘和总结的误区，曾经我是把这两个概念混淆的，实际上它们还是有些区别：总结主要是在一段任务结束后对成功或失败原因进行分析、归纳，更像是阶段性目标结果的整理报告；而复盘是通过对一段任务的整个过程进行全盘回溯，对这一过程进行再次模拟，思考每一步是否还有更好的方式，分析成功或失败的原因，通过假设推演，总结出更好的方案和不同的可能性。

那说了这么多，什么是复盘呢？接下来就讲讲我对复盘的理解。

什么是复盘

复盘，原是围棋术语，本意是对弈者下完一盘棋之后，重新在棋盘上把对弈过程摆一遍，看看哪些地方下得好，哪些下得不好，哪些地方可以有不同甚至是更好的下法等。 这个把对弈过程还原并且进行研讨、分析的过程，就是复盘。

可与注意到，其中提到了重新模拟一遍过程、分析好与不好、有哪些可以不同甚至提高的过程。那实际工作中，我们所指的复盘就是：

通过回顾自己、团队已做的事项，反思过程中可以改进的地方或者出现的问题和错误，分析造成这种情况的原因，沉淀行动过程中可以借鉴的优势，总结并学习经验教训，形成落地的行动，以此获得成长。

复盘的目的

那我们为为什么要进行复盘呢，或者说复盘的目的在于什么？

  • 反思一段时间犯过的错误和问题，定位出现的原因，寻找理想的解决改进方式，同样的问题只出现一次，避免再犯
  • 了结个人、团队的优劣势，帮助合理分配资源，让下一步行动目标更清晰
  • 知道风险点和如何控制风险，如何变得更好

简单来说，就是利用复盘寻找出发现了什么问题（what），并反思为什么会出现这种情况（why），最终总结经验教训怎么改进（how）。

什么时候进行复盘

那我们一般什么时候进行复盘，鉴于我自己的习惯是，有关于一个人的事项随时都可以进行复盘，关于团队的任务一般是在关键节点再进行复盘，因为团队人员多，虽然说复盘有一定好处，但是如果占用大家过多时间，也是会造成不好的效果。

因此我个人推荐的几个复盘时间点：

  • 出问题、犯错的时候
  • 任务里程碑、任务完成的时候
  • 定期复盘：每天、每周

首先，在出问题、犯错的时候及时进行复盘，能够有效的阻止问题进一步发展，迅速吸取经验，采取下一步的改善措施；在任务里程碑、任务完成时候进行复盘，则是可以对这一阶段的工作进行全盘的分析总结，寻找这一过程中的优劣势；对于定期复盘而言，可以根据个人和团队的习惯进行制定，选择适应的频率进行，辅助自己快速成长。

如何进行复盘

参考一些其他人的方法论和自己的实践，我倾向于按照下面这个步骤进行复盘：

  1. 回顾目标
    • 回顾当初行动的目的和意图
    • 明确我们最初要达到的目标和里程碑是什么
    • 我们预定的计划安排是怎样的
  2. 评估结果
    • 实际执行过程中发生了哪些情况
    • 是在什么情况下、如何发生的，引发的原因是什么
    • 与我们的既定目标相比，哪些做得好，哪些未达到预期
  3. 分析原因
    • 与实际预期有无差异
    • 如果有，为什么会出现这些差异，根本的原因是什么
    • 如果与预期相符，那么这个过程中成功的关键原因是什么
  4. 总结经验
    • 复盘整个过程，总结了哪些经验教训
    • 如果有同样的人做同样的事情，或是后续遇到同样的问题，应该怎么做，有哪些建议
    • 接下来我们应该怎么做，需要哪些新措施或是改善指出

简单来说，我们就是需要将整个过程中出现的、可能出现的问题，做的好和优势部分进行深度挖掘，寻找其根本原因，并进行总结归纳，形成可执行的行动和措施。

复盘的成功关键

一次成功的复盘能够给大家带来有价值的成长指导，那么确保复盘成功有几个关键点：

  • 三不
    • 不推卸责任
    • 不流于形式
    • 不迁怒于人
  • 四要
    • 氛围要 open，大家以学习的态度进行
    • 要刨根问底
    • 要找到自己的问题
    • 要将行动落地

复盘的数学公式推演

这部分是我们公司上讲到的，可以借鉴参考下

$$ a={ a }_{ 0 }{ (1+r) }^{ f } $$

其中 a(ability)是你一段时间后的能力水平，a0是你现在的能力水平，r 是你每次复盘提高的百分比，f 是你的复盘频率，因此，复盘可以让你呈复利式成长。当然这样说可能会比较理想，不过下面这张图我是比较认同的，复盘可以让损失变为经验教训，进而变成收益。
